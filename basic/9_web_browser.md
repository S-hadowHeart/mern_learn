A web browser is software used to access and view websites.  
Developers use browsers because they provide powerful built-in tools to inspect, debug, and optimize websites.

### Why developers use web browsers
- To view and test how their website looks and behaves
- To debug HTML, CSS, JavaScript using Developer Tools
- To monitor performance and fix issues
- To check network requests and responses
- To inspect security issues
- To analyze loading speed and improve SEO

---

## Browser Developer Tools

### Elements
Used to inspect and edit HTML and CSS. Shows DOM structure, allows live editing.

### Console
Used to run JavaScript commands and view logs, warnings, and errors.

### Sources
Shows all project files. Used for debugging JavaScript with breakpoints.

### Network
Shows all network requests (API calls, images, CSS, JS).  
Used to analyze loading speed, status codes, and request sizes.

### Performance
Used to record and identify performance bottlenecks such as slow scripts, layout shifts, rendering problems.

### Memory
Used to detect memory leaks, measure memory usage, and optimize large applications.

### Application
Shows storage-related data:
- LocalStorage
- SessionStorage
- Cookies
- IndexedDB
- Service Workers
- Cache

### Security
Displays certificate details, HTTPS issues, mixed content, etc.

### Lighthouse
Automated tool for:
- Performance analysis
- Accessibility review
- SEO scoring
- Best practices
