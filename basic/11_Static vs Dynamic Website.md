## Static Website
A static website contains fixed content that does not change unless manually updated.

### Advantages
- Fast loading speed
- Easy to build and host
- Cheap hosting
- Secure (no database involved)
- Great for small websites

### Disadvantages
- No real-time data
- No user interaction
- Hard to scale dynamic features
- Every update requires editing code manually

### When to use static websites
- Portfolio websites
- Blog without comments
- Information pages
- Landing pages
- Documentation sites

---

## Dynamic Website
A dynamic website shows content that changes based on user interaction or data from the server.

### Advantages
- Interactive
- Real-time updates
- Uses databases to store and fetch content
- Easier to scale and maintain
- Personalized user experiences

### Disadvantages
- Slower than static pages
- Needs more security
- Requires backend and database
- More expensive to host

### When to prefer dynamic websites
- Social networks
- E-commerce websites
- Dashboards
- Web apps (YouTube, Netflix)
- Any site needing login or database
